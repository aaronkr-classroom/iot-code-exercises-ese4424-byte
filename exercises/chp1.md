# chp1 연습문제

1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.<br>
  : 센서는 주변 환경의 온도, 습도, 빛 등의 정보를 감지하여 디바이스에 입력 데이터를 제공하는 장치이다.
반면 액추에이터는 디바이스의 제어 신호를 받아 LED 점등, 모터 회전 등 물리적인 동작을 수행하는 출력 장치이다.
즉, 센서는 입력(Input) 역할을 하고 액추에이터는 출력(Output) 역할을 한다.<br>

  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.<br>
  : 디바이스와 대시보드는 동작 환경과 목적이 서로 다르기 때문이다.
디바이스는 메모리와 처리 성능이 제한된 임베디드 환경에서 동작하므로 MicroPython이나 C와 같은 가벼운 언어를 사용한다. 반면 대시보드는 PC나 웹 환경에서 사용자 인터페이스와 데이터 시각화를 수행하므로 JavaScript, Python 등 화면 처리와 네트워크에 적합한 언어를 사용한다.
따라서 각 시스템의 성능, 기능, 실행 환경에 맞추기 위해 서로 다른 프로그래밍 언어를 사용한다. <br>

  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.<br>
: 브로커와 같은 정보 전달 전담 서버는 디바이스와 사용자(또는 대시보드) 사이에서 메시지를 중계하여 직접 연결 없이 통신할 수 있게 한다.
이를 통해 시스템 간 결합도를 낮추고, 여러 디바이스가 동시에 데이터를 주고받을 수 있어 확장성과 관리 효율이 향상된다. 또한 통신 안정성이 높아지고 네트워크 부하를 줄일 수 있는 장점이 있다. <br>

2. Q2
  - (c) (추가) Mosquitto와 Node-RED에 대해 설명하고, 실행에 적합한 컴퓨팅 환경을 제시하십시오.<br>
  
:Mosquitto는 MQTT 프로토콜을 사용하는 메시지 브로커로, IoT 디바이스와 서버 사이에서 데이터를 발행(Publish)하고 구독(Subscribe)하도록 중계하는 역할을 한다. 가볍고 안정적이어서 센서 데이터 전달에 널리 사용된다.
→ 적합한 환경: Linux 서버, Raspberry Pi, 저전력 PC 등 항상 실행 가능한 서버 환경

Node-RED는 웹 기반 플로우 프로그래밍 도구로, IoT 데이터 처리, 자동화 로직 구성, 대시보드 시각화를 쉽게 구현할 수 있다. 노드를 연결하는 방식으로 프로그램을 작성한다.
→ 적합한 환경: PC, 클라우드 서버, Raspberry Pi 등 웹 브라우저 사용이 가능한 컴퓨팅 환경

즉, Mosquitto는 데이터 전달(브로커) 역할을, Node-RED는 데이터 처리 및 시각화 역할을 수행한다.<br>
